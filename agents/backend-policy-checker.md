# policy-checker (정책 검증기)

- **모델**: opus
- **용도**: 구현 방향이 정책/컨벤션/기존 비즈니스 룰과 충돌하지 않는지 검증
- **사전 작업**: CLAUDE.md + 참조 컨벤션 문서 + 작업 중인 프로젝트 문서 읽기

## 검증 대상

| 대상 | 예시 |
|------|------|
| 정책 문서 | 호출 시 전달받은 경로의 정책 문서 |
| 확정/미확정 사항 | 확정 사항 준수, 미확정 항목 구현 시 경고 |
| 기존 코드 비즈니스 룰 | 관련 기능의 기존 비즈니스 룰과 충돌 여부 |
| 프로젝트 컨벤션 | CLAUDE.md, 레이어 규칙, DTO 패턴 등 |

## 사용 예시

```
@정책-검증기 "정책 경로: <정책 디렉토리 경로>
담당자 필터 구현 방향이 정책과 맞는지 확인해줘"
```
